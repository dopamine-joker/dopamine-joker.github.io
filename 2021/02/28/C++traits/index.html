C++traits

​ 在算法运用迭代器时，我们很可能会需要使用这个迭代器的相应型别(associated type)。

​ 相应型别其实就是其字面意思，它可以获取迭代器所指对象的类型。

5. Traits的更多应用

(这里的相应型别我个人理解是迭代器的属性描述)

迭代器的相应型别:

1.value type

所谓value type,就是指迭代器所指对象的类别。

2.difference type

difference type用来表示两个迭代器之间的距离。

3.reference type

引用类型，代表迭代器所指之物

4.pointer type

指针类型，代表迭代器所指之物的地址

5.iterator_category

根据移动特性与施行操作，迭代器被分为五类:

  • Input Iterator: 这种迭代器所指的对象，不允许外界改变。只读(read only)，并且它只允许单步向前，即只允许操作operator++
  • Ouput Iterator: 唯写(write only)，同样只允许operator++
  • Forward Iterator: 它可以在一个正确的区间上进行读写操作，它拥有Input Iterator的所有特性，和Output Iterator的部分特性，同样只允许operator++
  • Bidirectional Iterator: 在Forward Iterator的基础上增加了单步后退的能力，即可双向移动。支持operator++和operator—
  • Random Access Iterator: 该类迭代器能完成上面所有迭代器的工作，并且它涵盖了所有指针算能力，包括p+n,p-n,p[n],p1-p2等。

6.std::iterator的保证

​ 为了符合规范，任何迭代器的设计都应该提供五个内嵌相应类别，这样才有利于traits的萃取，也有利于与其他STL组件顺利搭配。

​ iterator class不含任何成员，纯粹只有型别定义，所以继承它不会有任何负担。由于后三个参数都有默认值，因此设计的新的迭代器只需要提供前两个参数即可。
